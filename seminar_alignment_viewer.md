# Alignment Viewer — セミナー説明資料

## 1. このアプリが解決する問題

ゲノムアセンブリの後、「どのscaffoldにリードが多く乗っているか」「リードの分布はどうなっているか」を確認したい。

従来の方法：
- コマンドラインで `samtools view` や `samtools tview` → 視覚的に把握しづらい
- IGV（デスクトップ版）を手動で起動、設定 → 毎回手間がかかる
- どのscaffoldを見ればいいかわからない → 手がかりがない

**→ このアプリは「BAMファイルをダブルクリックするだけで、重要な場所をすぐ確認できる」ブラウザビューアです。**

| 項目 | 内容 |

| 名前 | AlignmentViewer |

## 3. できること

### 3-1. scaffoldのランキング表示（サイドバー）

- **リード数の多い順**にscaffoldを自動でソートして左サイドバーに表示
- 各scaffoldのリード数とゲノム長を一覧表示
- クリックするだけで、そのscaffoldの**リード密集領域**にジャンプ

```
例（n1アセンブリ）:
  HiC_scaffold_3   76,439 reads  ████████████
  HiC_scaffold_5   36,571 reads  ████████
  HiC_scaffold_2   18,200 reads  ████
  ...（1,060 scaffolds 表示）
```

### 3-2. アライメントの可視化

- BAMファイルのリードをブラウザ上でリアルタイム表示
- 参照配列（リファレンスゲノム）の塩基も同時表示
- ミスマッチ・挿入・欠失の確認が可能

### 3-3. 座標指定ナビゲーション

- Start / End に座標を入力して任意領域にジャンプ
- **Peak ボタン**でそのscaffoldのリード密集点に自動ジャンプ
- ズームイン（+）・ズームアウト（−）

### 3-4. scaffoldの絞り込みフィルタ

- **テキスト検索**：scaffold名で絞り込み（例：`scaffold_924`）
- **Min-reads スライダー**：リード数が少ないscaffoldを非表示にして目的の候補を絞り込む

---

## 4. 処理の流れ

```
①  BAM + FASTA をアプリにドロップ
        ↓
②  自動セットアップ（Terminalで実行）
    - samtools idxstats で全scaffold のリード数を取得
    - リードが乗っているscaffoldの参照配列を抽出・整形
    - 各scaffoldのリード密集座標を計算 → JSON出力
        ↓
③  ローカルHTTPサーバー起動（localhost:8765）
        ↓
④  ブラウザが自動で開く → IGVビューア表示
```

**次回からは BAMをダブルクリックするだけ**（セットアップ済みの場合）。

---

## 5. 技術的な工夫

### リード密集座標の自動特定

単純に「scaffoldの先頭」ではなく、**100 kbビン単位でカバレッジを計算し、最もリードが集中している座標**を自動検出。全1,060 scaffoldで実際のリード位置をBAMから直接読み取るため、クリック一発で「見るべき場所」に飛べる。

### 大規模BAMへの対応

- 今回のBAM: **28,176 scaffold、403,994 reads**
- IGV.jsに `chromosomeOrder` を渡すことで、リードが乗っている1,060 scaffoldのみを処理対象に限定
- Range request（HTTP 206）対応サーバーでBAM・FASTAのランダムアクセスを実現

### 軽量なローカル運用

- インターネット不要（ライブラリはCDNから初回読み込みのみ）
- 機密データをクラウドにアップロードしない
- Python3標準ライブラリのみで動作するサーバー（依存なし）

## 6. 使用例・ユースケース

| ユースケース | 操作 |
|-------------|------|
| リードが多いscaffoldを素早く確認したい | サイドバー上位をクリック |
| 特定のscaffold（例：HiC_scaffold_924）を調べたい | 検索ボックスに `924` と入力 |
| リードが1本しかないscaffoldを除外したい | Min-readsスライダーで絞り込み |
| 特定の座標（既知の遺伝子座付近）を見たい | Start/End に座標を入力 → Go |
| アセンブリのエラー（ミスアセンブリ）を疑う領域を確認 | リードのスタック・方向を目視確認 |
